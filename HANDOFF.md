# Handoff — Portafolio Evan Morales

Portafolio personal de Evan Morales (Frontend / Full-Stack Developer), construido en Next.js + Tailwind a partir de un diseño importado de Claude Design.

**Repo:** https://github.com/EvanDavidMS/EvanDavidMS-portafolio

## Stack

- **Next.js 16** (App Router, TypeScript, Turbopack)
- **Tailwind CSS v4**
- **shadcn/ui** — componentes Tailwind + Radix (`Button`, `Badge`, `Card`, `Separator`, `Tooltip`, `Accordion`). No se usa MUI: se decidió explícitamente no mezclar dos sistemas de estilos distintos (ver sección de decisiones).
- **@icons-pack/react-simple-icons** — iconos de marca reales (React, Next.js, TypeScript, Node.js, etc.) para la sección de Skills.
- Fuentes: **Poppins** (texto general) + **Playfair Display italic** (acentos editoriales), vía `next/font/google`.

## Origen del diseño

El diseño visual fue importado desde un proyecto de Claude Design (`claude.ai/design`, proyecto "Portafolio Evan Morales", archivo `Portfolio Evan Morales.dc.html`) usando la herramienta `DesignSync`. Ese archivo `.dc.html` es un formato propietario de Claude Design (HTML + bindings `{{ }}` + un bloque de lógica de componente); se tradujo a mano a componentes React/Tailwind reales, no se copió el HTML literal.

    WavyBackground.tsx   — FONDO ACTIVO: ribbons animados grises→blanco en canvas full-bleed (client). ## Contenido de "Proyectos"

Los 11 proyectos, categorías, stack técnico, imágenes y links "Visita la página" se obtuvieron scrapeando `https://niocat.com/proyectos` y cada página de detalle (botón "Ver sitio en vivo"). ## Decisiones de diseño relevantes

- **Solo shadcn/ui, no MUI**: se preguntó explícitamente al usuario; shadcn/ui usa Tailwind + Radix (mismo sistema que ya tenía el proyecto) sin runtime de CSS-in-JS extra, mientras que MUI hubiera requerido Emotion y un theming paralelo. Se mantiene un solo sistema de estilos.
- **Tokens de shadcn re-mapeados** a la paleta exacta del diseño (`--background:#0A0A0A`, `--foreground:#F5F5F5`, `--border: rgba(255,255,255,.08)`, `--muted-foreground:#8A8A8A`, etc.) en `globals.css`, en vez de dejar los valores oklch por defecto — así cualquier componente shadcn nuevo hereda el look del sitio automáticamente.
- **"Sobre mí" se rediseñó por completo** (versión anterior: bio card + accordion "cómo trabajo" + code window + stats en cards + ilustraciones de globo/laptop). El usuario sintió que se veía "muy apilado" y que las ilustraciones no aportaban, así que se reemplazó por un split editorial simple: retrato grande (con marco offset detrás, puro CSS) a un lado, statement tipográfico grande + bio + stats en línea (separadas por `Separator` vertical, sin cards) al otro — mismo tono minimalista que el Hero. El contenido de "cómo trabajo" se integró como una frase dentro de la bio en vez de una lista/acordeón aparte.
- **Animaciones de scroll ("Sobre mí")**: `Reveal.tsx` sigue en uso — wrapper cliente que usa `IntersectionObserver` + las utilidades `animate-in fade-in slide-in-from-bottom-* fill-mode-forwards` de `tw-animate-css` (la librería de animación que trae shadcn) para que el retrato y el bloque de texto aparezcan con fade + slide al entrar en viewport. Ojo con el patrón: `fade-in` anima hacia la opacidad base del elemento, así que el estado "oculto" y el "animado" deben ser clases mutuamente excluyentes (no dejar `opacity-0` puesto a la vez que `animate-in`), si no la animación termina en opacidad 0.
- `Accordion` y `Card` (de shadcn) quedaron generados en `components/ui/` pero sin uso actual — se dejaron ahí por si se reutilizan en otra sección; no se importan en ningún componente ahora mismo.
- **Skills se rediseñó con iconos reales**: la lista plana de `techs` (strings) se reemplazó por `skillGroups` en `data.ts` — cada tecnología ahora tiene un componente de ícono de marca (`@icons-pack/react-simple-icons`) y una clase Tailwind literal `hoverText` (ej. `"group-hover:text-[#61DAFB]"`) para revelar el color de marca real al pasar el mouse. Los iconos se ven en gris neutro por defecto (consistente con el resto del sitio monocromático) y solo muestran su color al hacer hover — truco: el ícono recibe `color="currentColor"` y el color real se controla vía la clase `text-*` de Tailwind en el propio `<svg>`, así que el hover es 100% CSS, sin JS. Las tecnologías ahora están agrupadas por Frontend / Backend / DevX en vez de una lista plana + un bloque de texto redundante aparte.
  - Importante: las clases `group-hover:text-[#HEX]` deben escribirse como **string literal completo** en el código fuente (no construidas dinámicamente con template strings en runtime), porque Tailwind v4 escanea el texto de los archivos buscando nombres de clase completos — si se arma la clase con `` `group-hover:text-[${color}]` `` en runtime, Tailwind nunca la genera.
- ~~El sitio era **dark-only**~~ → ahora tiene **tema claro + oscuro con toggle** (ver Fase 2 en pendientes). Default oscuro. `:root` = claro, `.dark` = oscuro. Los colores de los componentes usan tokens (`--tint`, `--c-*`), no hex hardcodeado, para adaptarse al tema.
- **Aceternity UI — evaluado, dos componentes implementados**: se revisaron los 117 componentes de https://ui.aceternity.com/components y se armó una shortlist de cuáles calzan con el tono editorial/monocromático del sitio vs. cuáles chocarían (ej. 3D Globe, Vortex/Wavy/Aurora backgrounds compiten con el `ConstellationBackground` propio; ASCII/glitch text no calzan con el tono). Ninguno es un paquete npm — son patrones que se adaptaron a mano (igual que con shadcn) sobre nuestros componentes reales:
  - **Focus Cards** en Proyectos: `Projects.tsx` es client component con `useState<number | null>` para el índice hovereado — al pasar el mouse sobre una card, las demás bajan a `opacity-40 blur-[2px] scale-[0.98]` mientras la activa se eleva (`-translate-y-1`, borde más claro).
  - **Glowing Effect** en Skills: `GlowingEffect.tsx` es un componente client reutilizable — se renderiza como hijo de cualquier contenedor con clases `relative group`, y en un `useEffect` engancha su propio listener `mousemove` al `parentElement` (sin necesitar props de coordinación) para mover un radial-gradient vía custom properties CSS (`--x`/`--y`). El "anillo" de brillo se logra con el truco de `mask-composite: exclude` sobre dos capas de máscara (`content-box` vs `border-box` con `padding: 1px`), y aparece/desaparece con `group-hover:opacity-100` (sin JS para el fade). Ya está en cada tile de tecnología en Skills; queda disponible para reusar en Proyectos u otras cards si se quiere.
  - **Moving Border** en los CTA "Hablemos →" (Hero + Contact): `MovingBorderButton.tsx`. Frame de 1.5px con una luz blanca que gira sin fin (`@property --em-angle` + `conic-gradient` girando vía keyframe `emSpin`). En reposo el borde iguala el `rgba(255,255,255,0.22)` del diseño previo, así que se ve idéntico hasta que el destello barre el perímetro; el relleno blanco opaco tapa el centro para que la luz solo aparezca en el borde. Puro CSS, server component. Con `prefers-reduced-motion` cae a un frame estático. Ojo: `@property` requiere navegadores modernos (Chrome/Edge, Safari 16.4+, Firefox 128+); en uno viejo no gira pero se ve bien.
  - **Text Generate Effect** en el heading del Hero: `TextGenerateEffect.tsx`. Fade + de-blur palabra por palabra al cargar (keyframe `emWordIn` + `animation-delay` incremental por palabra). Puro CSS, server component. En vez de un string plano acepta `segments` (`{ text, className?, whole? }`) para preservar los acentos italic/underline del heading — las frases con estilo se pasan con `whole: true` para no romper el subrayado continuo al partir por palabras.
  - **Flip Words** en el rol del subtítulo del Hero: `FlipWords.tsx` (client). Cicla `["Frontend", "Full-Stack", "de producto", "de interfaces"]`; cada palabra entra con fade+blur+rise (keyframe `emFlipIn`). Truco: remontar el span vía `key={i}` re-dispara la animación de entrada, sin necesitar framer-motion/AnimatePresence. Enter-only (sin animación de salida). Con `prefers-reduced-motion` se queda fijo en la primera palabra.
  - **Spotlight** en el Hero: `Spotlight.tsx`. Haz de luz angular que entra una vez al cargar (keyframe `emSpotlight`; posición/rotación/escala viven en el keyframe — no agregar clases transform de Tailwind encima). Implementado como blob `radial-gradient` blureado (no el SVG pesado original), tuneado sutil y monocromo para superponerse al `ConstellationBackground` sin competir. El `<header>` del Hero ganó `overflow-hidden` para recortarlo.
  - **Comet Card** en Proyectos: combinado con el Focus Cards existente (decisión del usuario). `Projects.tsx` extrajo un subcomponente `ProjectCard`; la card activa se inclina en 3D hacia el cursor (`perspective` + `rotateX/rotateY`) con un glare radial que sigue al mouse. La rotación/glare se escriben en custom properties (`--rx/--ry/--mx/--my`) en `mousemove` (sin re-render); el lift/scale del Focus Cards se hornean en el mismo string `transform` para que ambos efectos compongan (no se puede mezclar `transform` inline con las utilidades translate/scale de Tailwind). El dim/blur/borde del Focus Cards siguen como clases desde el estado `hovered` del padre.
  - **Webcam Pixel Grid** — descartado a propósito: pide permiso de cámara al cargar (UX rara + privacidad) y choca con el tono editorial monocromático (misma razón que glitch/ASCII). Ver también `/blocks/text-animations`, que quedó cubierto por Text Generate + Flip Words.
  - Pendientes de la shortlist, no implementados aún: **Card Spotlight / Glare Card** (barrido de brillo extra en hover de Proyectos — ya hay glare del Comet Card, así que probablemente redundante), **Infinite Moving Cards** (marquee opcional de logos, en Footer o entre secciones).

- **Magic UI — 6 componentes agregados (a diferencia de Aceternity, estos SÍ son librería): se instalaron `motion` (framer-motion), `cobe` y `rough-notation`**. El código real se sacó de los registry JSON (`https://magicui.design/r/<name>.json`) y vive en `src/components/magicui/`. Reparto pensado para no sobre-animar:
  - **Text Animate** (`magicui/text-animate.tsx`, motion): en (a) los **eyebrows de sección** (About/Skills/Projects/Contact) — `by="character"`, `animation="blurInUp"`, `once`; y (b) los **párrafos de contenido** — About (intro + bio), Skills (descripción del stack), Contact (párrafo) — `by="word"`, `animation="blurInUp"`/`fadeIn`, `once`, entran en viewport. IMPORTANTE: Text Animate solo acepta `children: string`, por eso NO se usa en los headings con acentos italic/Highlighter (los rompería) — esos animan su subrayado con el Highlighter. El Hero se dejó sin tocar (ya tiene TextGenerateEffect + FlipWords).
  - **Highlighter** (`magicui/highlighter.tsx`, motion + rough-notation): subrayado hecho a mano (rough) en palabras clave — About "detalles"/"escala" y Skills "producción". `action="underline"`, `color="#8A8A8A"`, `isView` (se dibuja al entrar en viewport). Se usó underline monocromo en vez del highlight rosa default para respetar el tono.
  - **Globe** (`magicui/globe.tsx`, cobe + motion): **QUITADO de la UI por decisión del usuario** (ya no se monta en ningún lado; el archivo queda por si se reusa). Nota histórica: el bug de "el mundo no se ve" era CLIPPING (estaba sangrando fuera de su tile con `overflow-hidden`). Además, en captura **headless** de Playwright los marcadores salen pero el dot-map de continentes NO (limitación SwiftShader/ANGLE) — en navegador real con GPU sí. cobe v2: su tipo `COBEOptions` ya no declara `onRender`, se extendió con type local `GlobeOptions` + cast a `COBEOptions`.
  - **Testimonios — QUITADOS por completo** (al usuario no le gustaron): se eliminó `Highlights.tsx` (bento) y `TestimonialsBlock.tsx`, y se borró el array `testimonials` de `data.ts`. Contact volvió a "04". Queda sin usar: `magicui/animated-list.tsx`, `magicui/globe.tsx`, `magicui/morphing-text.tsx` (archivos presentes por si se reusan).
  - **Scroll-trigger de las animaciones** (Highlighter + Text Animate): se retrasó el disparo para que necesiten **más scroll** (el usuario lo sentía muy rápido). Ambos usan margen inferior `0px 0px -35% 0px` en el viewport/`useInView` → el elemento debe estar ~35% arriba del borde inferior para animar. Highlighter: `magicui/highlighter.tsx` `useInView({ margin: "0px 0px -35% 0px" })` + `animationDuration` default 900ms. Text Animate: `magicui/text-animate.tsx` `viewport={{ once, margin: "0px 0px -35% 0px" }}`. Verificado con Playwright: con el heading apenas entrando por abajo = 0 subrayados; centrado = 2.
  - **Fade a negro inferior**: en `page.tsx`, un `div` `fixed inset-x-0 bottom-0 h-[42vh] bg-gradient-to-b from-transparent to-[#0a0a0a]` (z-0, pointer-events-none) para que el `WavyBackground` resuelva limpio en la base del viewport (estética pedida por el usuario).
  - **Morphing Text** (`magicui/morphing-text.tsx`, sin deps — filtro SVG threshold + dos capas de texto blureadas; filtro por `style` inline porque la clase `filter-[...]` no resuelve bien en Tailwind v4): estaba en la sección Testimonios que se reemplazó por el bento, así que ahora mismo **el componente existe pero no está montado** en ningún lado. Queda listo para reusar (ej. un heading que morphea palabras).
  - **Progressive Blur** (`magicui/progressive-blur.tsx`, sin deps): tira `fixed` arriba (`position="top"`, z-40, debajo del navbar z-50) — el contenido se difumina progresivamente al pasar por detrás del navbar flotante. Capas apiladas de `backdrop-filter` con máscaras de gradiente.
  - **Webcam Pixel Grid / Animated Theme Toggler**: ver pendientes — el toggler necesita tema claro (Fase 2).

- **Timing / entradas afinadas (pedido del usuario):**
  - Hero: el reveal del heading (TextGenerateEffect) se hizo un poco más lento — `.animate-em-word` a `0.85s` (era 0.6s) + `stagger={0.17}` en `Hero.tsx` (era 0.12).
  - Hero — **entrada escalonada de todo**: badge / subtítulo / párrafo / botones ahora entran con fade + slide-up (`const enter = "animate-in fade-in slide-in-from-bottom-3 fill-mode-both duration-700 ease-out"` + `animationDelay` inline: 0.1s / 0.25s / 0.65s / 0.8s). El heading (TextGenerateEffect) recibió `delay={0.4}` (prop nueva) para arrancar después del subtítulo → cascada top-down. `fill-mode-both` evita el parpadeo. Verificado con Playwright (botones: opacidad 0 durante el delay, luego sube suave sin spike). El nav entra al final (1.6s).
  - Navbar: animación de entrada `.animate-em-nav` (keyframe `emNavIn`). **Espera 1.6s** (a que aparezca el hero) y luego hace un drop-in visible: `translateY(-30px) scale(0.95) → 0/1` con easing de rebote `cubic-bezier(0.34,1.56,0.64,1)`, 0.85s, `both` (el fill backwards lo mantiene oculto durante el delay). **OJO Tailwind v4**: `-translate-x-1/2` usa la propiedad CSS `translate` (no `transform`), así que el keyframe anima **solo `translateY`/`scale` vía `transform`** para no sumarse al -50% X y descentrar el nav. Verificado con Playwright (oculto durante el delay; centrado left=right=80 al terminar). Respeta `prefers-reduced-motion`.
  - Fondo: `WavyBackground speed` 0.0016 → **0.00184** (15% más rápido) en `page.tsx`.

- **Smooth scroll (Lenis)**: scroll suave con inercia en toda la página. `lenis` (dep nueva). `SmoothScroll.tsx` (client, montado en `layout.tsx`) inicializa Lenis (`lerp: 0.09`, `smoothWheel`, `anchors: true`) con su rAF loop; se desactiva bajo `prefers-reduced-motion` (scroll nativo). La instancia se comparte vía singleton `lib/lenis.ts` (`getLenis`/`setLenis`) para que `BackToTop` scrollee con `lenis.scrollTo(0)`. Anchors: Lenis intercepta los `<a href="#...">` y **ya respeta el `scroll-mt-[110px]`** de las secciones, así que NO se pasa offset (pasarlo duplicaba el gap del navbar → aterrizaba en 220 en vez de 110). CSS de Lenis en `globals.css` (`html.lenis { scroll-behavior: auto !important }` para que no pelee con el `scroll-behavior: smooth` nativo, que queda de fallback para reduced-motion). Lenis maneja el scroll real (window.scrollY), así que `useScrollState` (progreso/sección activa del navbar) y el `Reveal` on-scroll siguen funcionando. Verificado con Playwright (rueda: subida gradual con easing; anchors: aterrizan en 110).
